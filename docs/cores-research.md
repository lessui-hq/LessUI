### Key patterns in OnionOS core selection

**Mednafen dominates accuracy-focused emulation** for platforms like PC Engine, WonderSwan, Virtual Boy, Neo Geo Pocket, and Super Nintendo (via Supafaust). These cores prioritize compatibility over raw speed.

**Final Burn Alpha 2012 handles arcade emulation** for both Capcom CPS systems and Neo Geo, with the "2012" version optimized for lower-powered devices like the Miyoo Mini.

**BlueMSX serves double duty** as the default for both MSX/MSX2 and ColecoVision emulation, requiring the Databases and Machines folders from a BlueMSX installation for full functionality.

**The PORTS folder is unique** - it's not an emulator but a collection of native game ports (like Cave Story, Doom, and others) compiled specifically for the Miyoo Mini hardware.

All core names listed are without the `_libretro.so` suffix and `lr-` prefix - these are the base core identifiers as referenced in OnionOS configuration files and the GLO menu system.
